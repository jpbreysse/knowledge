# Plan: v2.2 — Connectors

## Context

Adds an admin-side `Connectors` registry where each row points to an external document store (URL + path prefix). On any asset's Documents tab, the user can then **link** a document from a connector — the link is stored as a reference (no file copy, no bytes), shows up alongside locally-uploaded files, and clicking the filename opens the external URL.

For v2.2 the only auth type in the dropdown is `none` — credential vaulting, OAuth, etc. are out of scope. One seeded connector (`document-store` at `http://localhost:3001/api`) ships with the demo. The page styling matches the sister ArborSpace · Knowledge app's Connectors page: green pill `enabled` badge, plain text `Edit · Delete` action links, monospace `NAME` column, page subtitle "HTTP integrations to other ArborSpace apps and external document stores."

None.

## Deliverables

### 1. Migration: `migrations/005_connectors.sql`

```sql
CREATE TABLE connector (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name        TEXT NOT NULL UNIQUE
                CHECK (name ~ '^[a-z0-9][a-z0-9-]{1,49}$'),
  label       TEXT NOT NULL,
  base_url    TEXT NOT NULL CHECK (base_url ~ '^https?://'),
  path_prefix TEXT,
  auth_type   TEXT NOT NULL DEFAULT 'none' CHECK (auth_type IN ('none')),
  enabled     BOOLEAN NOT NULL DEFAULT TRUE,
  created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX idx_connector_enabled ON connector (enabled);
CREATE TRIGGER trg_connector_set_updated_at
  BEFORE UPDATE ON connector FOR EACH ROW EXECUTE FUNCTION set_updated_at();

-- Extend asset_document to support external links.
ALTER TABLE asset_document
  ALTER COLUMN stored_path DROP NOT NULL,
  ALTER COLUMN mime_type   DROP NOT NULL,
  ALTER COLUMN size_bytes  DROP NOT NULL,
  ADD COLUMN connector_id UUID REFERENCES connector(id) ON DELETE RESTRICT,
  ADD COLUMN external_url TEXT,
  ADD CONSTRAINT asset_document_source_check CHECK (
    (stored_path IS NOT NULL AND connector_id IS NULL AND external_url IS NULL)
    OR
    (stored_path IS NULL AND connector_id IS NOT NULL AND external_url IS NOT NULL)
  );

CREATE INDEX idx_asset_document_connector_id ON asset_document (connector_id);
```

### 2. Drizzle mirror

Add `connector` to `src/lib/server/db/schema.ts`. Add the new nullable columns to `assetDocument`.

### 3. API routes

| Method | Path | Behaviour |
|---|---|---|
| `GET` | `/api/connectors` | List all, ordered by name. `?enabled=true` to filter. |
| `POST` | `/api/connectors` | Create. 201 / 409 on name conflict / 400 on validation. |
| `PATCH` | `/api/connectors/[id]` | Update any subset. |
| `DELETE` | `/api/connectors/[id]` | 204; 409 if referenced by `asset_document` (FK is RESTRICT). |

`POST /api/assets/[id]/documents` accepts JSON in addition to multipart:
- Multipart with `file=…` → existing local-upload path.
- JSON `{connector_id, external_url, filename}` → link path; `stored_path = NULL`, `connector_id` + `external_url` set.

`GET /api/assets/[id]/documents/[docId]` returns 404 if the doc is a link (the UI shouldn't link to that endpoint for linked docs anyway).

### 4. Validation

Add to `src/lib/server/validation.ts`:
- `validateConnector(body)` — `name` regex, `base_url` http(s), `label` 1–100, `path_prefix` optional starts with `/`, `auth_type ∈ ['none']`, `enabled` boolean.
- `validateLinkDocument(body)` — `connector_id` UUID + exists + enabled, `external_url` http(s), `filename` 1–255.

### 7. `/connectors/new` and `/connectors/[id]`

Same form on both, matches `/assets/new` and `/assets/[id]` pattern. Fields: `name` (slug; lowercase enforced client-side), `label`, `base_url`, `path_prefix` (placeholder `/api`), `auth_type` (dropdown — only `none` for v2.2), `enabled` (switch).

### 8. Asset detail — Documents tab additions

Above the existing upload form, add a second card: **Link from connector**.

Form fields:
- `Connector` dropdown (lists enabled connectors; disabled with "No connectors enabled" if list is empty)
- `URL` text input — paste the external URL (no browse picker in v2.2; that's v2.3)
- `Filename` text input — display name in the documents list
- `Link` button → POST JSON to `/api/assets/[id]/documents`

Documents list rows now distinguish source:
- **Uploaded files**: file icon, filename links to `/api/assets/[id]/documents/[docId]` (download), shows size + uploaded-at.
- **Linked docs**: link icon (`Link` from `@lucide/svelte/icons/link`), filename links to `external_url` with `target="_blank" rel="noopener"`, shows the connector's label as a small badge instead of size.

## Scope boundaries — do NOT build

- **No browse picker.** Paste URL by hand. Browse is v2.3 once the document app exposes a listing API.
- **No connection-test button.**
- **No real auth flows.** `auth_type` accepts only `none`.
- **No file copy / import.** Linking only.
- **No bulk import.**
- **No connector audit log.**
- **No CORS / network probes** of the connector base URL.

## Quality expectations

- The mixed Documents list (uploaded + linked) renders in one round-trip with a left-join on `connector` for the connector label.
- The new CHECK on `asset_document` makes inconsistent rows impossible (no row with both `stored_path` and `connector_id`, no row with neither).
- `npm run check` and `npm run build` clean — no new warnings.
- Existing 50 MB upload limit and MIME sniffer for the local-upload path are not touched.

## Verification checklist

1. Migration applies cleanly.
2. Seed creates the `document-store` row; re-running is a no-op.
3. `/connectors` shows one row matching the seed.
4. POST `/api/connectors` with duplicate name returns 409.
5. POST with `{name: 'BAD!'}` returns 400 (slug regex).
6. PATCH `{enabled: false}` flips the badge from green to muted.
7. DELETE on the seeded connector with no linked docs succeeds (204), then attempt with a linked doc returns 409.
8. On `/assets/[id]` Documents tab: Link form posts JSON, the new linked doc appears with link icon and `Document Store` badge, click opens the external URL in a new tab.
9. Existing local-upload still works.
10. The download endpoint returns 404 for a linked doc id.
11. `npm run check` clean; `npm run build` clean.
